## What

Checks that frontmatter includes a `description` field and that it stays within the agentskills.io max (**1024 characters**).

## Why

The description is the **trigger**. Agents load name + description before the body. Missing or oversized descriptions hide the skill or waste always-on context. Vague copy (“helps with documents”) never fires; overly broad copy hijacks every request.

## How to fix

1. Add a `description` that states **what** the skill does and **when** to use it.
2. Prefer concrete triggers over slogans.

```yaml
# weak
description: API helper

# stronger
description: >-
  Use when writing code that calls the Gemini API for text generation,
  multi-turn chat, image generation, or streaming.
```

3. If you need more than ~1k characters, move detail into the body or `references/` — do not pad the description.
4. Pair with [R014](/reference/rules/R014) / [R025](/reference/rules/R025) so the *content* of the description is trigger-ready, not only present.

## What

Scans every string value in the skill's YAML frontmatter for `<` or `>` characters.

## Why

Angle brackets in frontmatter can be parsed as tags and inject unintended instructions when metadata loads into a system prompt. This rule is locked because that creates a prompt-injection risk.

## How to fix

Remove `<` and `>` from frontmatter values, or move that content into the Markdown body where it is not treated as metadata.

## What

Checks that a skill's `scenarios.yaml` file can be parsed and has the expected scenario structure.

## Why

Scenarios define expected trigger behavior. Invalid scenario data cannot be used to evaluate whether the skill activates in the right situations.

## How to fix

Correct the YAML syntax and required fields in `scenarios.yaml`.

Include at least:

- one case the skill **should** handle
- one case it **should not** fire (negative / ignore)
- optionally one edge case

Each scenario should name a success criterion you can grade. Coverage of those cases is judged under [R027](/reference/rules/R027).

Pick one path. After a permanent install, `dora` and `doraval` are the same binary.

## Homebrew (recommended, macOS)

```bash
brew tap saif-shines/tap && brew trust saif-shines/tap && brew install doraval
```

Upgrade later with `brew upgrade doraval` or `dora update`.

## npm (cross-platform)

```bash
npm install -g @hacksmith/doraval
```

Ships a prebuilt binary per platform (macOS arm64/x64, Linux x64/arm64, Windows x64). Node ≥ 14.18.

## Bun

```bash
# permanent
bun add -g @hacksmith/doraval

# one-shot
bunx @hacksmith/doraval
```

## One-shot (no install)

```bash
npx @hacksmith/doraval
# or
bunx @hacksmith/doraval
```

Useful for trying doraval or CI without a global install. Prefer a permanent install for day-to-day use.

## Alpine / musl

Platform binaries target glibc. On Alpine/musl, run from source with Bun:

```bash
bunx @hacksmith/doraval
```

## For coding agents

Agents and CI usually skip a permanent install and run the CLI on demand with `npx` (no global install, always the latest):

```bash
npx @hacksmith/doraval review . --ci
```

To make a coding agent reach for `dora` on its own, install the doraval skill:

```bash
npx skills add saif-shines/doraval
```

This drops the doraval skill into your agent's skill directory (`.claude/skills/`, `.codex/skills/`, `.cursor/rules/`, and the like). The agent then verifies any skill, plugin, rule, or agent config with `dora review` before reporting the work done. If `dora` is not on `PATH`, the skill falls back to `npx @hacksmith/doraval`. See [For coding agents](/for-agents/) for how the skill drives the CLI.

## Verify

```bash
doraval --version
# same binary:
dora --version
```

## Common mistakes

- Global install succeeds but `dora` is missing: ensure your npm/Bun global bin directory is on `PATH`.
- Alpine or musl hosts: use Bun (`bunx` / `bun add -g`), not the glibc platform binary.

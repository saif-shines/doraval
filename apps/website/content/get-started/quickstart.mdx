```bash
npx skills add saif-shines/doraval
```

Check agent context on **this project** — skills, rules, surfaces, next fixes:

```bash
npx @hacksmith/doraval
# same as: dora   (after npm / brew / bun install)
```

## What you get

Bare `dora` is read-only. No API key. No writes.

1. **Agent surfaces** — which agents look configured  
2. **Health** — skill / manifest issues with short reasons  
3. **Next** — numbered follow-ups (`dora fix …`, `dora review …`)

```text
$ dora

  Health
    ✓ .claude/skills/review    valid
    ✗ .claude/skills/deploy    Missing "description"

  Next
    1. dora fix .claude/skills/deploy
    2. dora review --all
```

## Optional next steps

Only if the scan (or your work) needs them:

| Goal | Command |
| --- | --- |
| Apply mechanical fixes | `dora fix .` (diffs first; `--yes` for agents/CI) |
| Grade a skill or the repo | `dora review .` · CI: `dora review --all --quick --ci` |
| Scaffold a skill / rule / plugin | `dora new` or `dora new skill --for claude --name … --yes` |
| Keep a principle | `dora memory add "…" --weight 8` |

Full tables: [Command reference](/commands/scan/). JSON and CI: [Use with your agent](/for-agents/).

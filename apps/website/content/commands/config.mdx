`dora config` reads and writes doraval settings. Use it to wire a review judge (CLI or API), pick a model, or script configuration without editing files by hand.

```bash
dora config                 # interactive hub (TTY)
dora config setup           # judge wizard: provider, model, backend
dora config get
dora config get eval.model
dora config set eval.model gpt-4.1
dora config set eval.judge auto
```

Common keys: `eval.model`, `eval.provider`, `eval.judge`, `agent.command`. Run `dora config get` for the full table (secrets masked in human output; use `--format json` for raw values).

On a TTY, bare `dora config` opens an interactive hub: list keys, edit a value (enums for provider and judge), or run setup. Prefer `get` and `set` in scripts.

## Common mistakes

- Expecting `dora review --deep` to work without a judge: run `dora config setup` first (or set `eval.judge` / provider keys).
- Pasting API keys into chat logs: set them via config or environment variables; human `get` masks secrets.

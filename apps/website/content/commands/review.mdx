`dora review` grades skills, memory principles, and related artifacts. Run it after scaffold or before merge. Tiers that can run do run; LLM and sessions skip when unavailable unless you require them with `--deep` or `--sessions`.

```bash
dora review [PATH]
```

`PATH` is a skill directory or project root (default `.`).

## Tiers

1. **Structure**: parseable manifests, required fields, path consistency
2. **Heuristics**: dead refs, duplicates, keyword-level memory violations, `scripts/` security smells (outbound calls, secret prompts)
3. **LLM**: judgment when a judge model is configured
4. **Sessions**: mines real agent session logs (Claude, Grok, Cursor, Codex, Copilot) for evidence the skill was actually used

Project principles from [memory](/concepts/memory/) apply as a rubric. A violated principle weighted **7+** is an error, not a warning. Details: [Review tiers](/concepts/review-tiers/).

## Options

| Flag | Meaning |
| --- | --- |
| `--quick` | Tiers 1–2 only (no LLM) |
| `--deep` | Require LLM tier; exit `2` if no judge |
| `--sessions` | Require the session-evidence tier; exit `2` if no recent sessions |
| `--all` | Review every artifact under the path |
| `--fail-on error\|warning` | Exit `1` trigger (default `error`) |
| `--format table\|json` | Output format |
| `--ci` | Machine mode (implies JSON) |
| `--cwd <path>` | Working directory override |

## Exit codes

| Code | Meaning |
| --- | --- |
| `0` | Clean |
| `1` | Issues found (per `--fail-on`) |
| `2` | Could not run (for example `--deep` with no judge, or `--sessions` with no logs) |

## Examples

```bash
dora review ./skills/deploy
dora review --quick .
dora review --deep ./skills/deploy
dora review --all --quick --ci
dora review CLAUDE.md
```

CI structural gate (no LLM):

```bash
dora review --all --quick --ci
```

## Common mistakes

- Using `--deep` in CI without a judge: exit code is `2`, not “issues found.” Prefer `--quick` for structural gates.
- Assuming skipped tiers mean a full pass: without a judge, LLM findings never appear. Configure a judge with `dora config setup` when you need judgment.
- Expecting review to rewrite files: it reports. Use `dora fix` for mechanical fixes.

`dora providers` prints a static packaging and spec matrix: manifest paths, marketplace layout, and MCP filename per agent. Use it when scaffolding or shipping packages, not when diagnosing the current repo.

```bash
dora providers
dora providers --json
```

**Not** “which agents does this repo use?” That is bare [`dora`](/commands/scan/) (Agent surfaces).

Scaffold with `dora new --for <agent>`.

## Common mistakes

- Using `providers` instead of `dora` to see local configuration: wrong tool for repo diagnosis.

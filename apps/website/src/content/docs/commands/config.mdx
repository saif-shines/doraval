`dora config` reads and writes `~/.doraval/config.yml` using dot-notation keys.

## Usage

```bash
dora config set <key> <value>
dora config get [key]
```

## config set

```bash
dora config set eval.model glm-4-flash
dora config set eval.provider zai
dora config set eval.judge auto
```

Values are coerced by type: `true`/`false` become booleans, numeric strings become numbers, everything else stays a string.

## config get

```bash
# Get a single key
dora config get eval.model

# Get a nested object
dora config get eval

# Print entire config (YAML)
dora config get
```

## Common keys

| Key | Description | Example values |
|-----|-------------|----------------|
| `eval.model` | Model used for judge calls | `glm-4-flash`, `gpt-4o-mini` |
| `eval.provider` | Provider name | `zai`, `openai`, `groq`, `openrouter` |
| `eval.judge` | Judge invocation mode | `auto` (default), `api`, `cli` |
| `eval.base_url` | Custom OpenAI-compatible endpoint | `https://your-proxy.example.com/v1` |
| `eval.timeout_ms` | Judge API timeout (ms) | `180000` (3 min), `300000` for slow reasoning models |
| `agent.command` | Default coding agent CLI | `claude`, `codex` |

## Supported providers

| Provider | Env var | Built-in base URL |
|----------|---------|-------------------|
| OpenAI | `OPENAI_API_KEY` | `https://api.openai.com/v1` |
| Z.ai / GLM | `ZAI_API_KEY` | Zhipu endpoint |
| Groq | `GROQ_API_KEY` | Groq endpoint |
| OpenRouter | `OPENROUTER_API_KEY` | OpenRouter endpoint |
| Anthropic | `ANTHROPIC_API_KEY` | OpenAI-compatible endpoint |
| Custom | any | `eval.base_url` |

## Judge mode

```bash
dora config set eval.judge auto    # API if key present, CLI fallback (default)
dora config set eval.judge api     # Direct API only — error if no key
dora config set eval.judge cli     # Force coding agent CLI
```

## Custom endpoint

```bash
# Permanent
dora config set eval.base_url https://your-proxy.example.com/v1

# Per-run override
OPENAI_BASE_URL=https://your-proxy.example.com/v1 dora judge ./my-skill/
```

## Interactive setup

For first-time configuration, the guided flow is easier:

```bash
dora init          # full one-time setup
dora evals setup   # configure only the judge LLM
```

See [evals setup](/commands/evals-setup/) for the interactive flow.

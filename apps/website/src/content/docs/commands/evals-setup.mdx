`dora evals setup` lets you pick or change the LLM vendor for eval judging without re-running the full `dora init` flow.

## Usage

```bash
dora evals setup
```

## What it does

Walks an interactive wizard with three prompts:

1. **Which LLM vendor?** — select from built-in providers or choose Custom.
2. **API key** — skipped if a matching env var is already set (e.g. `ZAI_API_KEY`). Otherwise stored in `~/.doraval/config.yml`.
3. **Which model?** — fetches a live model list from the provider's `/v1/models` endpoint after the key is resolved. Provider defaults appear at the top marked *recommended*. Falls back to hardcoded defaults if the fetch fails.

The result is written to your existing `~/.doraval/config.yml` under `eval.provider` and `eval.model`.

## Supported vendors

| Vendor | Env var | Recommended model |
|--------|---------|-------------------|
| **OpenAI** | `OPENAI_API_KEY` | `gpt-4o-mini` |
| **Z.ai / GLM** | `ZAI_API_KEY` | `glm-4-flash` |
| **Groq** | `GROQ_API_KEY` | `llama-3.1-8b-instant` |
| **OpenRouter** | `OPENROUTER_API_KEY` | `anthropic/claude-3-haiku` |
| **Anthropic** | `ANTHROPIC_API_KEY` | `claude-3-5-haiku-20241022` |
| **Custom** | any | — (you provide base URL) |

## Security note

Prefer setting the API key in your environment (`ZAI_API_KEY`, `OPENAI_API_KEY`, etc.) rather than storing it in `config.yml`. The wizard skips the key prompt entirely when a matching env var is detected.

## Prerequisites

Requires an existing doraval config (run `dora init` first).

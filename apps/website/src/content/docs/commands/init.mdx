One-time setup for decision memory (the journal) and agent integration so the journal can remember decisions and principles while you scale AI context through skills, plugins, and more.

## Usage

```bash
doraval init [options]
```

## What it does

`init` performs several one-time steps. The interactive CLI now clearly labels them:

1. **Decision memory (journal)** – set up the persistent memory that remembers decisions and principles while scaling AI context.
2. **Agent integration** – configure the coding agent doraval will use to help capture and enrich journal entries.
3. **Evaluation setup** – optional model so you can verify agents are following your decisions/principles as you scale.

Details:

- Ensures the GitHub CLI (`gh`) is available and authenticated.
- Prompts for (or accepts via flags/env) your journal repository — this is the persistent decision memory that remembers principles while scaling AI context.
- Prompts for a project name (defaults to the current directory).
- Configures a coding agent command and prompt template so that `doraval journal add` can automatically enrich entries with tags and rationale.
- Sets up evaluation so you can verify agents are following your decisions and principles as you scale via skills and plugins.
- Creates local cache directories under `~/.doraval/`.
- Fetches the relevant journal files from GitHub.

## Options

| Flag           | Description                                                                 |
|----------------|-----------------------------------------------------------------------------|
| `-r, --repo`   | Journal repository (owner/name). Smart default from git remote or gh account. Env: `DORAVAL_JOURNAL_REPO` |
| `-p, --project`| Project name (defaults to basename of current directory)                    |
| `--refresh`    | Re-fetch journal files even if the project is already registered            |

## Non-interactive usage

```bash
doraval journal init --repo yourname/yourname.md --project my-project
```

Or with environment variables:

```bash
DORAVAL_JOURNAL_REPO=yourname/yourname.md \
DORAVAL_PROJECT=my-project \
doraval init
```

This sets up the journal that will remember decisions and principles as you scale AI context.

## After init

Your journal is now ready to remember decisions and principles while you scale AI context through skills, plugins, etc.

Use the journal commands:

- `doraval journal list`
- `doraval journal add "..."` (agent can help enrich)
- `doraval journal update`
- `doraval journal sync`

See the [Agent Journal](/concepts/agent-journal/) concept page.

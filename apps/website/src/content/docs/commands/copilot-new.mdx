`dora copilot new` is a context-aware wizard that scaffolds a new skill or plugin for GitHub Copilot, following its packaging conventions.

```bash
dora copilot new
dora copilot new my-skill
dora copilot new --yes --intent distribute my-plugin
```

The interactive flow and outcomes are identical to [`dora claude new`](/commands/claude-new/) — context detection, intent selection (self / self-later / distribute), and scaffold. The difference is the manifest location:

- **Plugin path**: creates `.github/plugin.json`
- Marketplace listing: `marketplace.json`

## Options

| Flag | Description |
|------|-------------|
| `--yes` | Skip interactive prompts (use defaults and flags) |
| `--intent` | `self` \| `self-later` \| `distribute` |

## After scaffolding

```bash
dora validate . --for copilot
dora validate . --for copilot:plugin
```

See [`dora claude new`](/commands/claude-new/) for the full flow description and outcome examples.

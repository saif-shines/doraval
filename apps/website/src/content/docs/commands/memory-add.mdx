Add a principle to project memory. It is written immediately under `~/.doraval/memory/` — no staging step, no separate publish.

```bash
dora memory add "Never use default exports"
dora memory add "Run bun test before commit" --weight 8
dora memory add "Prefer named exports" --weight 3 --tags style
dora memory add "Shared house rule" --global --body "Applies to every project."
```

## Weight guide

| Weight | Meaning |
|--------|---------|
| **8–10** | Hard rule — surfaces as an error in `dora review` |
| **5** | Default — normal principle |
| **1–3** | Soft preference |

## Flags

| Flag | Description |
|------|-------------|
| `--weight <1-10>` | Importance (default `5`) |
| `--tags <a,b>` | Comma-separated tags |
| `--body <text>` | Rationale or detail |
| `--global` | Store in global memory (all projects) |

## Examples

```bash
# Hard rule for this repo
dora memory add "Always run dora review before shipping skills" --weight 8

# Soft preference
dora memory add "Prefer async/await over .then()" --weight 3 --tags style
```

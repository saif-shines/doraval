`eval` and `evals` are aliases for [`judge`](/commands/judge/) when given a skill path. All three produce identical output.

```bash
dora eval  ./skills/my-skill/   # → judge
dora evals ./skills/my-skill/   # → judge
dora judge ./skills/my-skill/   # primary name
dora evals setup                # configure judge LLM (only on evals)
```

See the **[judge documentation](/commands/judge/)** for full usage, options, JSON envelope format, and configuration.

## Session-based evaluation

The old `dora eval` session-based adherence checker (interactive session picker, per-session tool-call analysis) is now [`dora drift`](/commands/drift/):

```bash
dora drift ./skills/my-skill/   # was: dora eval --skill ...
dora drift                       # repo sweep (all skills × all sessions)
```

## Quick reference

| Old command | New command | Behavior |
|-------------|-------------|----------|
| `dora eval --skill <path>` | `dora judge <path>` | Artifact vs rubric (session-free) |
| `dora eval` (session picker) | `dora drift <path>` | Session behavioral analysis |
| `dora judge <path>` | unchanged | Same command, now primary |
| `dora eval history` | — | Not available (stored results coming) |

## Configuration

`eval` shares judge LLM configuration. Set up once:

```bash
dora evals setup     # configure vendor + model
dora init            # full one-time setup (includes evals)
```

See [evals setup](/commands/evals-setup/) for supported vendors and config keys.

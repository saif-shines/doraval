**doraval** is a CLI that validates structure and measures rubric drift for AI agent skills.

It works with any agent following the [Agent Skills spec](https://agentskills.io/specification) — Grok, Claude Code, Cursor, Windsurf, and others.

## Why

AI agent skills are Markdown files with YAML frontmatter, structured steps, code examples, and guardrails. They look simple but are hard to keep consistent:

- A missing `name` field means the agent can't discover the skill
- Vague phrasing like "you might want to" confuses agents that need direct instructions
- No code examples leaves agents guessing at syntax
- No `MUST` / `MUST NOT` constraints means unpredictable behavior

Manual review doesn't scale. doraval automates these checks.

## Three commands

| Command | What it does | Speed |
|---------|-------------|-------|
| `skill validate` | Structural correctness — frontmatter, required fields, file layout | Instant |
| `skill drift` | Rubric deviation — 6 categories of quality drift | Instant |
| `skill judge` | AI-driven qualitative assessment via LLM | *(coming soon)* |

`validate` and `drift` are deterministic and run locally with zero network calls. `judge` will require an LLM API key.

## Quick start

```bash
bun install -g doraval

doraval skill validate ./my-skill/
doraval skill drift ./my-skill/
```

Or run without installing:

```bash
bunx doraval skill validate ./my-skill/
```

## Beyond skills

`doraval journal` provides a separate system for recording project decisions and principles with pushback (see [Agent Journal](/concepts/agent-journal)). It is completely optional and lives outside your project directories.

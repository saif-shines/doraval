**Scale your AI context for coding agents.** Make your next context work (skills, plugins & more) for your team, community, or self. Context engineering toolkit for AI coding agents.

Whether you are optimizing context for yourself, your team, or a wider community, the job is the same: make skills, plugins, and decisions work on the first attempt — not after days of debugging.

[Quickstart →](/get-started/quickstart/)

## Name and pronunciation

**doraval** is pronounced *dor-uh-val* — stress the last syllable, like **Dora** + **eval**.

The **Dora** comes from [Doraemon](https://en.wikipedia.org/wiki/Doraemon), the robot cat who pulls the right gadget for the job. The **-val** comes from **evaluation** — three tiers that go from static checks to real session behavior. `validate` checks the document (including static quality heuristics); `judge` scores the artifact against a rubric; `drift` checks reality in real transcripts. The shorter alias **`dora`** is the same CLI.

## The problem

- Give 10 engineers (or agents) a skill. Only 3/10 succeed on the first try.
- Publish a plugin to the community? Most adopters hit the same slow ramp.
- Decisions drift across sessions because nothing persists.
- You cannot confidently say "this works for Claude, Cursor, and Copilot."

doraval **left-shifts success**: catch breakage before the first session, not after the third debugging thread.

## What doraval does

| Command | What it scales | Speed |
|---------|----------------|-------|
| `validate` | Catch broken skills, plugins, hooks, MCP (plus static skill heuristics) before anyone uses them | Instant |
| `judge` | Artifact quality vs a best-practices rubric (one LLM call; `eval` / `evals` aliases) | Seconds |
| `drift` | Proof that agents followed your skills in real session transcripts | Minutes |
| `claude new` / `cursor new` etc. | Scaffold resources that load correctly across agents | Instant |
| `init` | One-time setup for decision memory (journal) + agent integration | Instant |
| `journal` | Decisions that persist across sessions and people | Instant |
| `bump` / `config` | Semver manifests and dot-notation config | Instant |
| `ui` | Local dashboard for capturing context without retyping | Instant |
| `providers` | See which agents understand which packaging | Instant |

`validate` is the workhorse. Point it at a directory or GitHub URL and it tells you what is broken before you scale context to anyone else. See [three-tier verification](/concepts/three-tier-verification/) for how `validate`, `judge`, and `drift` fit together.

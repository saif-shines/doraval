**Scale your AI context for coding agents.** Make your next context work (skills, plugins & more) for your team, community, or self.

Whether you are optimizing context for yourself, your team, or a wider community, the job is the same: make skills, plugins, and decisions work on the first attempt — not after days of debugging.

[Quickstart →](/get-started/quickstart/)

## Name and pronunciation

**doraval** is pronounced *dor-uh-val* — stress the last syllable, like **Dora** + **eval**.

The **Dora** comes from [Doraemon](https://en.wikipedia.org/wiki/Doraemon), the robot cat who pulls the right gadget for the job. The **-val** comes from **evaluation**. The shorter alias **`dora`** is the same CLI.

## The problem

- Give 10 engineers (or agents) a skill. Only 3/10 succeed on the first try.
- Publish a plugin to the community? Most adopters hit the same slow ramp.
- Decisions drift across sessions because nothing persists.
- You cannot confidently say "this works for Claude, Cursor, and Copilot."

doraval **left-shifts success**: catch breakage before the first session, not after the third debugging thread.

## What doraval does

| Command | What it scales |
|---------|----------------|
| `dora` / `scan` | See agent surfaces, skill health, contradictions, next actions |
| `review` | Structure + heuristics + optional LLM / sessions |
| `fix` | Apply mechanical fixes (diff + confirm) |
| `new --for <agent>` | Scaffold skills, rules, agents, plugins |
| `memory` | Principles that persist; enforce in review; promote to AGENTS.md |
| `reconcile` | Settle contradictions between agent configs |
| `bump` / `config` / `sessions` | Version manifests, config, session drill-down |
| `providers` | Which agents understand which packaging |

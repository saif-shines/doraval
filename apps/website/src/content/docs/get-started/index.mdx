**doraval** is a context engineering toolkit for coding agents. It validates the files that shape how your agents behave — skills, plugins, hooks, MCP configs, memory files, and more.

Works with Claude Code today. Cursor, Codex, and Windsurf support is coming.

## Why

The context you give coding agents — SKILL.md files, CLAUDE.md, plugin manifests, hooks, MCP configs — is what determines whether they're useful or chaotic. These files are easy to write but hard to keep correct:

- A missing `name` field means the agent can't discover the skill
- A malformed `plugin.json` silently breaks the entire plugin
- An `@import` pointing at a deleted file creates a silent hole in memory
- Unknown hook event names are silently ignored
- Vague phrasing like "you might want to" confuses agents that need direct instructions

Manual review doesn't scale. doraval automates these checks across every artifact type your agent consumes.

## Commands

| Command | What it does | Speed |
|---------|-------------|-------|
| `validate` | Auto-detect and validate all agent config in a directory | Instant |
| `skill validate` | Structural correctness for a single skill | Instant |
| `skill drift` | Rubric deviation — 6 categories of quality drift | Instant |
| `skill judge` | AI-driven qualitative assessment via LLM | *(coming soon)* |

`validate` is the primary command. It auto-detects which validators apply (skill, plugin, marketplace, hooks, MCP, subagent, command, memory) and runs them all. It also accepts Git URLs — point it at a GitHub repo and it clones, validates, and cleans up.

## Quick start

```bash
bun install -g doraval

dora validate .                                     # local directory
dora validate https://github.com/obra/superpowers   # remote repo
dora validate . --for claude:plugin                 # specific validator
dora skill drift ./my-skill/                        # rubric drift
```

Or run without installing:

```bash
bunx doraval validate .
```

## Beyond validation

`doraval journal` provides a separate system for recording project decisions and principles with pushback (see [Agent Journal](/concepts/agent-journal)). It is completely optional and lives outside your project directories.

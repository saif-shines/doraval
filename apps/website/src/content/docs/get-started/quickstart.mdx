Scale context for **yourself**, **your team**, or **your community**. Same workflow whether you are tuning your own agent, onboarding juniors, or publishing a plugin.

This quickstart assumes doraval is installed. If not, see [Installation](/get-started/installation/).

```
  Scale AI context (you, team, or community)
                    │
                    ▼
             ┌─────────────┐
             │  dora new   │  scaffold skill / rule / plugin
             └─────────────┘
                    │
                    ▼
             ┌─────────────┐
             │ dora review │  structure + heuristics (+ LLM)
             └─────────────┘
                    │
                    ▼
             ┌─────────────┐
             │ dora memory │  principles that stick
             │ add · list  │
             │ promote     │
             └─────────────┘
                    │
                    ▼
        context that works first try
```

<Steps>

1. ### Scan what you already have

   ```bash
   dora
   ```

   Bare `dora` lists agent surfaces, skill health, contradictions, and numbered next actions.

2. ### Scaffold context

   ```bash
   # Interactive: type → agent → name
   dora new

   # Or non-interactive
   dora new skill --for claude --name my-context --description "Team context" --yes
   dora new plugin --for claude --name my-plugin --yes
   ```

   Prefer `dora new --for <agent>`. Thin wrappers (`dora claude new`, …) still work. See [claude new](/commands/claude-new/).

3. ### Review before anyone relies on it

   ```bash
   dora review .
   dora fix .                 # mechanical fixes; always shows a diff
   ```

4. ### Record principles that stick

   ```bash
   dora memory add "Use dora review before shipping any skill change" --weight 8
   dora memory list
   dora memory promote        # optional: write hard rules into AGENTS.md
   dora memory sync           # optional: private git backup
   ```

   For Claude SessionStart hooks:

   ```bash
   dora memory context --json --quiet
   ```

   See [memory add](/commands/memory-add/), [memory sync](/commands/memory-sync/), and [Memory](/concepts/memory/).

5. ### Optional deeper checks

   Session adherence and multi-agent reconcile when you need them:

   ```bash
   dora sessions
   dora reconcile
   ```

</Steps>

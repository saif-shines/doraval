doraval is designed to run in CI. Use `--format json` and `--ci` for pipeline-friendly output.

## GitHub Actions

```yaml
name: Validate Skills
on: [push, pull_request]

jobs:
  lint-skills:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - run: bun install -g doraval

      - name: Validate
        run: doraval skill validate ./skills/ --ci

      - name: Drift check
        run: doraval skill drift ./skills/ --ci
```

## Flags for CI

| Flag | What it does |
|------|-------------|
| `--ci` | Machine-friendly output, non-zero exit codes on issues |
| `--format json` | Structured JSON to stdout — pipe to `jq` or consume programmatically |

## Exit codes

- `validate` exits with code **1** when errors are found
- `drift` exits with code **1** when drift is detected (with `--ci`)

## JSON output

Both commands write structured JSON when `--format json` is set:

```bash
doraval skill validate ./my-skill/ --format json --ci
doraval skill drift ./my-skill/ --format json --ci
```

Pipe to `jq` for filtering, or consume directly in your pipeline scripts.
